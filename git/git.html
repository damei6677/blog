2.6 git diff

修改文件或者添加一个文件后，git diff命令是比较修改或提交的文件内容，命令输出当前目录中修改的内容，并不包含新加的文件。这些内容还没有添加到本地缓冲区。

当执行git add * 命令后，将修改添加到本地缓冲区，在执行git diff –cached查看缓冲区中与上次提交前之间的差别。

执行git commit -m “update code”后，git diff和git diff –cached都不会有任何输出了。
